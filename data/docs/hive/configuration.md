# HIVE Configuration

HIVE is configured through `dashboard.config.json`, generated by the setup wizard. The file is gitignored — each developer has their own. A committed `dashboard.config.example.json` serves as a reference template.

## Configuration File

### Core Settings

```json
{
  "project": "my-app",
  "title": "My App — H.I.V.E.",
  "projectsDir": "C:\\Users\\you\\Projects",
  "port": 3333,
  "logDir": ".hive/logs"
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `project` | string | — | Project identifier |
| `title` | string | `"Dev Dashboard"` | Browser tab title |
| `projectsDir` | string | — | Base directory where repos live |
| `port` | number | `3333` | Dashboard HTTP port |
| `logDir` | string | `".hive/logs"` | Directory for service log files |

### Repositories

```json
{
  "repos": ["hive", "my-api", "my-web", "my-db"]
}
```

An array of directory names within `projectsDir`. Each is monitored for git status, branch changes, and commit activity.

### Services

```json
{
  "services": {
    "web": {
      "label": "Web",
      "port": 8080,
      "startCmd": "npm run dev",
      "repoDir": "my-web"
    },
    "api": {
      "label": "API",
      "port": 3000,
      "startCmd": "npm run start:dev",
      "repoDir": "my-api"
    }
  }
}
```

Each service gets a log streaming widget. The dashboard detects whether the service is running by checking if its port is in use (TCP connect check). Start/stop/restart controls call the configured `startCmd`.

### External Monitors

```json
{
  "externalMonitors": [
    {
      "key": "prod-api",
      "label": "Production API",
      "url": "https://api.myapp.com/health",
      "type": "http",
      "interval": 30,
      "alarm": true
    },
    {
      "key": "statuspage",
      "label": "Azure Status",
      "url": "https://status.azure.com",
      "type": "statuspage",
      "interval": 60,
      "alarm": false
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `key` | string | Unique identifier |
| `label` | string | Display name |
| `url` | string | URL to check |
| `type` | `"http"` or `"statuspage"` | Check method |
| `interval` | number | Seconds between checks (default: 30) |
| `alarm` | boolean | Enable audio alarm on failure |

### Azure DevOps

```json
{
  "ado": {
    "org": "my-org",
    "project": "My Project",
    "team": "My Team",
    "users": ["Developer A", "Developer B"],
    "prRepos": ["my-api", "my-web"],
    "workItemTypes": ["Bug", "User Story", "Feature"],
    "activeStates": ["Active", "New"],
    "pipelineIds": [1, 2, 3]
  }
}
```

Requires `ADO_PAT` environment variable set in `.env`.

### GitHub

```json
{
  "github": {
    "org": "my-github-org",
    "users": ["ghuser1", "ghuser2"],
    "prRepos": ["org/repo1", "org/repo2"],
    "watchRepos": ["org/repo1"]
  }
}
```

Requires `GITHUB_TOKEN` environment variable set in `.env`.

### Sentry

```json
{
  "sentry": {
    "org": "my-sentry-org",
    "projects": ["frontend", "backend"]
  }
}
```

Requires `SENTRY_AUTH_TOKEN` environment variable set in `.env`.

### Releases

```json
{
  "releases": {
    "repoDir": "my-web",
    "path": "data/releases.json"
  }
}
```

Points to a JSON file containing release version history. The releases widget reads this file from the specified repo directory.

### Environment Diff

```json
{
  "envDiff": {
    "local": { "repoDir": "my-api", "path": ".env" },
    "dev": { "repoDir": "deployments", "path": "dev-settings.env" },
    "qa": { "repoDir": "deployments", "path": "qa-settings.env" },
    "prod": { "repoDir": "deployments", "path": "prod-settings.env" }
  }
}
```

Each entry maps a label to a file path (relative to a repo within `projectsDir`). The Env Diff widget compares them side by side.

### Documentation

```json
{
  "docsDir": "data/docs",
  "docsSkipDirs": [".git", "node_modules", ".obsidian", ".trash"]
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `docsDir` | string | `"data/docs"` | Path to markdown documentation |
| `docsSkipDirs` | string[] | `[".git", "node_modules", ".obsidian", ".trash"]` | Directories to exclude from the tree |

### Other Settings

```json
{
  "swaggerUrl": "http://localhost:3000/swagger-json",
  "dbRepo": "my-db",
  "cliRepo": "my-cli",
  "bookmarks": [
    { "label": "Swagger", "url": "http://localhost:3000/swagger" }
  ],
  "quickActions": {
    "seed-db": { "repoDir": "my-db", "cmd": "knex", "args": ["seed:run"] }
  }
}
```

## Data Directories

HIVE uses two data directories:

| Directory | Default | Contains | Purpose |
|-----------|---------|----------|---------|
| **Data dir** | `{hive}/data/` | metrics, layouts, databases, db-scripts | Shared project data |
| **API dir** | `{hive}/data/api/` | collections, environments | API client data |
| **Docs dir** | `{hive}/data/docs/` | Markdown files | Documentation |
| **Private dir** | `~/.config/hive/data/` | user prefs, private collections, history | Per-user data |

Override data and private directories in your config:

```json
{
  "dataDir": "/custom/path/to/data",
  "privateDataDir": "~/my-hive-data"
}
```

Paths starting with `~` expand to the user's home directory. Relative paths resolve from `projectsDir`.

## Environment Variables

Set these in `{hive}/.env` (gitignored):

```env
ADO_PAT=your-azure-devops-personal-access-token
GITHUB_TOKEN=your-github-personal-access-token
SENTRY_AUTH_TOKEN=your-sentry-auth-token
PORT=3333
```

## Settings UI

All configuration can also be edited through the Settings tab in the dashboard itself. Changes are persisted to `dashboard.config.json` in real time.
