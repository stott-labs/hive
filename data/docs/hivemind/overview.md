# Hivemind — Overview

**Hivemind** is a shared Claude Code configuration package. Install it once and every repo on your machine gets the same set of AI-powered slash commands — `/prd` for product requirements, `/create-pr` for pull requests, `/create-bug` for bug tracking, and more.

## The Problem It Solves

Without Hivemind, every repository needs its own Claude Code configuration. PR conventions, bug templates, planning workflows — they all live per-repo or not at all. When a team member runs `/create-pr`, the behavior should be the same whether they're in the API repo, the web repo, or the database repo.

Hivemind centralizes this. One repo defines the team's AI workflows. Each developer clones it once, runs setup, and every Claude Code session automatically loads the shared skills.

## How It Works

```
Developer's Machine
├── ~/.config/hivemind/
│   ├── config.md          # Org-specific values (generated by setup)
│   └── paths.env          # Runtime path discovery
├── ~/.claude/
│   └── skills/            # Symlinks → hivemind/.claude/skills/*
│
├── Projects/
│   ├── hivemind/          # This repo (cloned once)
│   │   ├── .claude/skills/  # Skill definitions
│   │   ├── setup.sh/ps1    # Interactive setup
│   │   └── CLAUDE.md       # Platform instructions
│   ├── my-api/            # ← All repos get hivemind skills
│   ├── my-web/            # ← via the --add-dir alias
│   └── my-db/             # ←
```

1. **Skills are symlinked**, not copied. Running `git pull` in the hivemind repo updates all skills across the system immediately.
2. **Configuration is local**. The repo is org-agnostic. Org names, ADO project IDs, reviewer lists — all of it lives in `~/.config/hivemind/config.md`, which is gitignored.
3. **Provider-flexible**. One codebase supports both Azure DevOps and GitHub. Skills check the `provider` field at runtime.

## Key Features

| Feature | Description |
|---------|-------------|
| **10 slash commands** | Planning, PRs, bugs, repos, scheduling, and more |
| **Provider abstraction** | Same commands work with ADO (via MCP) or GitHub (via `gh` CLI) |
| **Symlink distribution** | Skills update with `git pull` — no reinstall needed |
| **Shared config** | Team conventions in one place, org-specific values per developer |
| **HIVE integration** | Optional dashboard pairing for notifications and launching |
| **Platform CLAUDE.md** | Shared instructions loaded into every repo's Claude context |

## Tech Stack

| Component | Technology |
|-----------|-----------|
| Setup | Bash + PowerShell (cross-platform) |
| Skills | Markdown files with YAML frontmatter |
| Config | YAML-style Markdown (`key: value`) |
| ADO integration | Azure DevOps MCP |
| GitHub integration | `gh` CLI |
| Notifications | HTTP POST to HIVE dashboard |
